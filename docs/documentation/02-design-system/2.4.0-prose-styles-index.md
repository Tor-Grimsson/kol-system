## Choosing the Right Spec

1. **Need a class name?** Check `2.4.1` first—it lists every `kol-*` style plus usage guidance.
2. **Rendering CMS content?** Start with `2.4.2` for the Stack wrappers, then layer on documentation or Málrómur specifics if required.
3. **Building wiki/docs pages?** Use `2.4.3` for exact px/rem measurements plus the baseline code example extracted from `/foundry/prose-specs/documentation`.
4. **Editorial serif layouts?** `2.4.4` mirrors `/foundry/prose-specs/malromur` with the full H1–H6 stack, captions, pull quotes, and list behavior.

## Implementation Checklist

- **Wrapper Selection:** Pick `.kol-prose`, `.kol-prose-wide`, or `.kol-prose-compact` based on the column width (65/90/45ch). Reference `2.4.2`.
- **Typography Class:** Apply the matching `kol-*` utility (see `2.4.1`) when you need to enforce a specific scale outside of the prose wrappers.
- **Content Variant:** 
  - Documentation: use `.doc-*` classes defined in `packages/ui/css/docs.css` and documented in `2.4.3`.
  - Málrómur: use `.malromur-*` classes from the Foundry specimen (documented in `2.4.4`).
- **CMS Mapping:** When wiring Portable Text or MDX content, map headings/lists/marks to the Stack wrappers first, then layer variant-specific styles via class overrides.
- **Design Tokens:** Never substitute hex values—colors come from tokens defined in `packages/ui/theme.css`, typography tokens from `2.2.x`, and spacing from `2.3.x`.
